oracle.txt (原始元数据)
这是未经清洗的原始记录文件，包含了方法的详细签名信息。

文件格式： 非结构化文本。

数据结构： [类全名] [方法详细签名] [位置编码]

位置编码格式： e{起始偏移量}:{长度}

示例数据：
CH.ifa.draw.util.StorableOutput public void writeStorable(CH.ifa.draw.util.Storable) e1044:114;

特点：
保留了完整的 Java 方法签名（修饰符、返回类型、参数类型），有助于处理重载方法（Overloading）。
使用“长度”而非“结束位置”来标记代码块。




oracle_refined.txt (清洗后的结构化数据)
这是为了便于程序（如 Python 脚本或 Java 解析器）批量处理而生成的 CSV 文件。

文件格式： CSV（逗号分隔值）。

数据结构： class_name,function_name,offset_start,offset_end

位置坐标格式： 绝对起始位置, 绝对结束位置

示例数据：
CH.ifa.draw.util.StorableOutput,writeStorable,993,1102（

特点：
移除了复杂的签名细节，仅保留方法名。
将坐标转换为物理文件的绝对起止点（Start & End），更适合直接在文件流中截取代码片段。
数据经过校准（例如去除了空白或注释干扰），精度更高。
